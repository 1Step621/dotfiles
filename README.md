# dotfiles

This is my configuration repository.
It uses [Den](https://den.denful.dev/), an aspect-oriented, context-driven Nix configuration framework.

## Customization

Customize the following files to match your preferences and hardware.

- `modules/aspects/(user).nix`: User-specific configuration.
  - `modules/aspects/my/password.nix`: Stores the user's hashed password.
- `modules/aspects/(host).nix`: Hardware-specific configuration.
- `modules/hosts.nix`: Defines which users are enabled on each host.

## Apply the configuration

```bash
# Bootstrap nh if it is not installed yet
nix run .#hostname -- switch

# if nh is available
nh os switch . # NixOS
nh darwin switch . # Darwin
nh home switch . # Home Manager
```

## Test in a VM

```bash
nix run .#vm-hostname
```

## Generate an architecture diagram

```bash
nix run .#diagram
```

## Edit flake.nix

`flake.nix` is generated by `vic/flake-file`. Do not edit `flake.nix` directly. Please edit `modules/dendritic.nix` and run `nix run .#write-flake`.

## Use as a flake input

You can use this flake as your own flake input. For more information, see the [Den documentation](https://den.denful.dev/guides/namespaces/).

```nix
{ inputs, ... }: {
  # Import "my" namespace from this flake, merging with local definitions
  imports = [ (inputs.den.namespace "my" [ inputs.onestep-dotfiles ]) ];
}
```
